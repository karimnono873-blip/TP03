1. Contexte Historique et Théorique

La Programmation Orientée Objet (POO) : Apparue dans les années 1960 avec le langage Simula, la POO a révolutionné la conception logicielle en modélisant les problèmes sous forme d'objets interagissant entre eux. Le langage Java, créé par Sun Microsystems en 1995, a popularisé ce paradigme grâce à sa robustesse et sa portabilité ("Write Once, Run Anywhere").

Le Jeu du Saute-Mouton : Ce casse-tête classique trouve ses racines dans les jeux de société solitaires traditionnels. Le but est de transposer deux groupes d'éléments (ici des pions rouges et noirs) de part et d'autre d'un plateau en utilisant un espace vide central. La résolution mathématique de ce jeu repose sur la théorie des graphes et l'anticipation des états de blocage.

2. Solution de l'Exercice (Code Java)

Le code suivant a été développé en respectant strictement l'arborescence algorithmique demandée par le cours (Initialiser, Afficher, Jouer un coup, Fin de jeu, Rejouer) ainsi que les contraintes de déplacement.

import java.util.Scanner;

public class SauteMouton {
    // Constantes et Variables globales requises par l'enseignant
    static final int NbMAX = 10;
    static char[] Plateau;
    static int nbPions;
    static int indVide;

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        boolean rejouer = true;

        while (rejouer) {
            initialiserPlateau(scanner);
            boolean fini = false;

            while (!fini) {
                afficherPlateau();
                jouerCoup(scanner);
                fini = finDeJeu();
            }

            System.out.print("Voulez-vous refaire une partie ? (O/N) : ");
            char rep = scanner.next().toUpperCase().charAt(0);
            if (rep != 'O') rejouer = false;
        }
        scanner.close();
    }

    static void initialiserPlateau(Scanner scanner) {
        do {
            System.out.print("Entrez le nombre de pions (max " + NbMAX + ") : ");
            nbPions = scanner.nextInt();
        } while (nbPions <= 0 || nbPions > NbMAX);

        Plateau = new char[2 * nbPions + 1];
        
        for (int i = 0; i < nbPions; i++) Plateau[i] = 'N';
        
        indVide = nbPions;
        Plateau[indVide] = ' ';
        
        for (int i = nbPions + 1; i < 2 * nbPions + 1; i++) Plateau[i] = 'R';
    }

    static void afficherPlateau() {
        System.out.println("\nPlateau de jeu :");
        for (int i = 0; i < Plateau.length; i++) {
            System.out.print("[" + Plateau[i] + "]");
        }
        System.out.println();
        for (int i = 0; i < Plateau.length; i++) {
            System.out.print(" " + i + " ");
        }
        System.out.println();
    }

    static void jouerCoup(Scanner scanner) {
        System.out.print("Position du pion à jouer : ");
        int pos = scanner.nextInt();

        if (pos < 0 || pos >= Plateau.length || pos == indVide) {
            System.out.println("Position invalide.");
            return;
        }

        char pion = Plateau[pos];

        // Logique stricte dictée par l'algorithme du TP
        if (pion == 'R') {
            if (pos - 1 == indVide) {
                echanger(pos, indVide);
            } else if (pos - 2 == indVide && Plateau[pos - 1] == 'N') {
                echanger(pos, indVide);
            } else {
                System.out.println("Coup interdit pour le pion Rouge.");
            }
        } else if (pion == 'N') {
            if (pos + 1 == indVide) {
                echanger(pos, indVide);
            } else if (pos + 2 == indVide && Plateau[pos + 1] == 'R') {
                echanger(pos, indVide);
            } else {
                System.out.println("Coup interdit pour le pion Noir.");
            }
        }
    }

    static void echanger(int pos1, int pos2) {
        char temp = Plateau[pos1];
        Plateau[pos1] = Plateau[pos2];
        Plateau[pos2] = temp;
        indVide = pos1; // Mise à jour de l'index de la case vide
    }

    static boolean finDeJeu() {
        // Vérification de la victoire
        boolean gagne = true;
        for (int i = 0; i < nbPions; i++) {
            if (Plateau[i] != 'R') gagne = false;
        }
        if (gagne) {
            System.out.println("Bravo ! Configuration finale gagnante atteinte !");
            return true;
        }

        // Vérification du blocage
        boolean bloque = true;
        for (int i = 0; i < Plateau.length; i++) {
            if (i == indVide) continue;
            if (Plateau[i] == 'R' && (i - 1 == indVide || (i - 2 == indVide && Plateau[i - 1] == 'N'))) {
                bloque = false; break;
            }
            if (Plateau[i] == 'N' && (i + 1 == indVide || (i + 2 == indVide && Plateau[i + 1] == 'R'))) {
                bloque = false; break;
            }
        }

        if (bloque) {
            System.out.println("Situation de blocage. Vous avez perdu.");
            return true;
        }
        return false;
    }
}

4. Explications et Mécanismes du Code

L'implémentation a été conçue pour refléter la méthodologie séquentielle et modulaire exigée[cite: 100]. Chaque sous-programme répond à une exigence spécifique :

  • Déclaration des Variables Globales : Conformément au sujet du TP[cite: 136, 137, 138, 139], nous avons déclaré NbMAX comme constante, Plateau comme tableau de caractères, et géré les indices avec nbPions et indVide.
  • Méthode initialiserPlateau() : Elle sécurise l'entrée utilisateur pour s'assurer que le nombre de pions ne dépasse pas la limite imposée[cite: 140, 141]. Ensuite, elle alloue dynamiquement l'espace mémoire requis (2 * nbPions + 1) et place les 'N' à gauche, 'R' à droite, avec un espace vide au centre.
  • Méthode jouerCoup() : C'est le cœur logique du programme. Elle traduit le pseudo-code fourni dans l'énoncé de manière stricte.
    • Pour un pion Rouge ('R') : Il vérifie si la case à sa gauche directe (pos - 1) est vide, ou s'il doit sauter un pion Noir (pos - 2 vide et pos - 1 == 'N')[cite: 102, 103, 107].
    • Pour un pion Noir ('N') : La logique est inversée (vérification vers la droite : pos + 1 ou pos + 2)[cite: 108, 109, 112].
  • Méthode echanger() : Cette fonction utilitaire accomplit le "Déplacement = échange de valeurs" stipulé dans les contraintes[cite: 113], simplifiant la lisibilité de jouerCoup().
  • Méthode finDeJeu() : Elle analyse le tableau de manière linéaire. D'abord, elle vérifie si les pions 'R' occupent les premières positions (Victoire). Ensuite, elle scanne tous les pions restants pour s'assurer qu'au moins un déplacement est encore valide (Sinon : Blocage)[cite: 130, 131].
